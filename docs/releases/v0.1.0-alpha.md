# CodexButler v0.1.0-alpha

This is the first Android alpha for local-first Codex thread monitoring and approval handling.

## What Works

- Connect to a CodexButler backend on the same LAN, VPN, or authenticated private tunnel.
- View active, pinned, and recent threads grouped by project.
- Open a thread and read the message timeline with newest messages at the bottom.
- Create backend-owned Codex threads from the phone.
- Send follow-up prompts to idle threads.
- Review and answer pending approval requests.
- Store the backend bearer token in platform secure storage when available.

## Requirements

- A trusted Codex host running the CodexButler backend.
- A long random `BACKEND_AUTH_TOKEN`.
- `CODEX_MOCK_MODE=true` for first UI testing, then `CODEX_MOCK_MODE=false` with `CODEX_CONNECTION_MODE=child` for real Codex testing.
- Phone and backend on the same trusted network path.

Do not expose port `4545` directly to the public internet. Use LAN, VPN, or an authenticated private tunnel.

## Known Limitations

- Desktop-owned approvals only work if `CODEX_CONNECTION_MODE=proxy` can attach to a compatible local Codex app-server control socket.
- The supported real approval flow is for turns started through CodexButler's backend app-server connection.
- Codex Desktop may need refresh or restart before it shows turns created through CodexButler.
- The attached APK is a release variant signed with the default debug key for alpha side-loading, not a Play Store release build.

## Smoke Test

1. Install the APK on Android and allow installation from the selected source.
2. Start the backend with `CODEX_MOCK_MODE=true`.
3. In Settings, set `http://<host-lan-ip>:4545` and the bearer token.
4. Tap Test connection.
5. Open the mock idle thread and send a short prompt.
6. Run `pnpm mock:approval` on the host.
7. Verify the approval appears on the phone and can be approved or denied.

## Security Notes

- Never publish `.env`, bearer tokens, tunnel secrets, Codex credentials, or backend logs containing private paths.
- Rotate `BACKEND_AUTH_TOKEN` after sharing an alpha build with testers who should no longer have access.
- Keep destructive, networked, migration, publish, and write-heavy commands behind explicit approval.
- The Android manifest is expected to request only network access plus framework-level non-exported receiver permissions for this alpha.
- Android backup is disabled so the stored backend token is not opted into app data backup.
